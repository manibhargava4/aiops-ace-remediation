How it works.

The six-stage self-healing loop the site dramatizes — and actually runs.

  1. Buggy release ships. A hung-thread ESQL flow (uncapped WHILE loop on a poison message) is built by CI and deployed.
  2. The thread hangs. The flow thread never yields; process CPU climbs toward a full core.
  3. Prometheus notices. A PromQL rule crosses the alert threshold; Alertmanager fires a webhook.
  4. AI writes the RCA. A FastAPI triage service reads logs + metrics + ESQL source and an LLM (Ollama / Claude / Bedrock) writes the root-cause analysis and a fix — filed as a GitHub issue.
  5. The fix ships. The corrected flow redeploys through real CI/CD — Jenkins (push) or GitHub Actions → ArgoCD (pull GitOps).
  6. The gate decides. A validation gate checks a hard numeric CPU threshold first (LLM verdict advisory) and confirms recovery.
