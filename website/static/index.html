AIOps Incident Remediation Pipeline — detect → diagnose (AI) → fix → verify

A production incident lifecycle, fully automated: a buggy IBM ACE-style flow pegs a CPU, Prometheus alerts, an AI triage service performs root-cause analysis with an LLM (Ollama locally, Claude via Anthropic API/Bedrock in the cloud), the fix is redeployed, and an AI validation gate verifies recovery from live metrics.

Terraform · EKS · ECR · IRSA · OIDCPrometheus · Grafana · Alertmanager GitHub Actions · Trivy · CheckovFastAPI · Docker Compose Claude / Ollama (pluggable)

Architecture

[1] Buggy ACE app — hung-thread ESQL bug · built by GitHub Actions → image in ECR → deployed to EKS (Terraform-provisioned) · locally: Docker Compose
│ poison message → WHILE loop never exits → CPU pegs at 1 core
[2] Prometheus detects the CPU spike → Grafana dashboard shows it
│ rate(cpu[1m]) > 0.8 for 30s (3m in prod)
[3] Alertmanager → AI Triage Service (FastAPI) — pulls container logs + CPU series + ESQL source · LLM produces RCA + proposed fix → GitHub issue + local evidence store (S3 on AWS)
│ LLM_BACKEND = ollama | anthropic | bedrock
[4-5] Fix merged → same pipeline — build → Trivy scan → ECR → EKS deploy via OIDC (no long-lived keys)
│
[6] AI Validation Gate — hard threshold first (post-deploy CPU < 0.3), LLM verdict advisory → pass/fail on the PR
