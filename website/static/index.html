the loop · 06 stages · scroll →

Six stages.
Zero pages at 3am.

01
ship

Buggy release ships

The ACE simulator deploys with a hung-thread ESQL bug via Docker Compose — the same manifests deploy to minikube through Jenkins (push) or ArgoCD (pull).

GitHub Actions builds the image, Trivy gates CRITICAL/HIGH CVEs, pushes to ECR and deploys to EKS — authenticated by a short-lived OIDC token, not a stored key.

02
hang

The thread hangs

A poison message — AccountId = NULL — enters a WHILE loop with no exit. The flow thread spins forever. One core pegs at 100%. In production this was a 3am page; here it's reproduced deterministically, every run.

03
detect

Prometheus notices

rate(process_cpu_seconds_total[1m]) > 0.8 holds for 30 seconds — the alert flips from pending to firing and Alertmanager webhooks the triage service.

rate(container_cpu_usage_seconds_total[2m]) > 0.8 for 3 minutes — kube-prometheus-stack evaluates it, Alertmanager routes it to the triage pod.

04
diagnose

AI writes the RCA

The triage service collects the container logs, the CPU series and the actual ESQL source, and a local LLM (qwen3:8b via Ollama — free, offline) writes the root-cause analysis with a proposed fix as a diff. Filed as a GitHub issue.

Same evidence, bigger brain: Claude via Amazon Bedrock. The pod's ServiceAccount maps to an IAM role (IRSA) scoped to Bedrock + the S3 evidence bucket — zero keys in the cluster.

05
fix

The fix ships

Merging the fix redeploys through the pipeline you choose below — watch Jenkins push it, or watch a git commit become a deployment as ArgoCD pulls it.

The fix PR merges → build → scan → ECR → EKS rollout. The pipeline is the only path to production; nobody kubectl-applies by hand.

06
verify

The gate decides

Post-deploy CPU must be < 0.3 cores — a hard, deterministic threshold. The LLM verdict rides along as advisory color. The AI is never the only gate — the design sentence this whole project exists to demonstrate.

two pipelines · two philosophies

Push, or pull?

push-based

Jenkins

The pipeline holds cluster credentials and runs kubectl apply itself. Build → Trivy → local registry → minikube. Classic, imperative, auditable in the build log.

the catch: CI holds prod credentials, and manual drift goes unnoticed.

pull-based · gitops

Actions + ArgoCD

CI never touches the cluster — it builds, scans, pushes to GHCR and commits the new image tag to git. ArgoCD, inside minikube, reconciles the cluster to match git. Deployment = commit.

drift is detected & self-healed. rollback is a git revert.

Jenkins needs ci/ up · GitOps needs ArgoCD on minikube — see ci/README.md. jenkins ui ↗
